# Pixel Art Levels 51–100: Design Spec

## Overview

Add 50 new handcrafted-quality pixel art levels (51–100) to Pixel Power. Each level depicts a recognizable subject — animals, buildings, or holiday themes — using an expanded 12-color palette. Levels are generated via an automated pipeline that uses the Claude API to produce pixel art grids, then validates solvability with the existing solver.

## Goals

- Every level reveals recognizable pixel art (not random patterns)
- Subjects are mixed across categories so gameplay stays varied
- Difficulty ramps via a sub-1.0x ammo multiplier that forces return-queue mastery
- The entire level set is generated by a single CLI command

## Difficulty Curve

### Ammo Multiplier

Linear descent from 1.00x at level 51 to 0.85x at level 100:

```
multiplier = 1.00 - ((level - 51) / 49) * 0.15
```

- Level 51: 1.00x (exact ammo)
- Level 65: 0.957x
- Level 75: 0.927x
- Level 85: 0.896x
- Level 100: 0.85x (15% ammo deficit — must recycle pigs via return queue)

### Grid Size Progression

- Levels 51–60: 8×8 to 10×10
- Levels 61–80: 9×9 to 11×11
- Levels 81–100: 10×10 to 12×12

### Colors Per Level

No hard cap. The art dictates the palette. Most levels use 3–5 colors.

## Milestones

No new milestone system. Every level's pixel art reveal IS the reward.

## Generation Pipeline

### Architecture

```
Subject Config → Claude API → Puzzle Builder → Solver Validator → Level Output
```

### Step 1: Subject Config

File: `tools/generator/pixel-art-subjects.json`

JSON object keyed by level number. Each entry contains:
- `subject` — what to draw (e.g. "Fish")
- `category` — "animal", "building", or "holiday"
- `grid` — [rows, cols]
- `colors` — allowed color palette for this level
- `description` — art direction for Claude (e.g. "A simple tropical fish facing right")

### Step 2: Claude API Pixel Art Generation

File: `tools/generator/pixel-art-prompt.js`

Calls the Claude API with a structured prompt for each subject. The prompt:
- Specifies exact grid dimensions
- Lists the allowed color strings
- Describes the subject and orientation
- Requests output as a strict JSON 2D array
- Uses `null` for empty/background cells
- Includes art guidelines: recognizable silhouette, use all listed colors, fill the design area

Response is parsed as JSON and validated to match the expected grid dimensions.

### Step 3: Bench & Ammo Calculation

From the generated layout:
1. Count cells per color
2. Calculate ammo multiplier for the level number
3. Compute total ammo per color: `ceil(count * multiplier)`
4. Split ammo across pigs (max 8 ammo per pig)

### Step 4: Solvability Validation

Uses the existing `tools/generator/solver.js`:
1. Tests 2000+ random pig orderings
2. If solvable → accept
3. If unsolvable → retry strategy:
   - Bump ammo by +1 on the tightest color (up to 3 bumps)
   - If still unsolvable → re-generate pixel art via Claude
   - Max 5 total attempts per level
   - If all attempts fail → flag for manual review

### Step 5: Output

File: `src/data/pixel-art-levels.js`

Exports an array of level objects matching the existing format:
```js
{
  id: string,         // kebab-case (e.g. 'fish')
  name: string,       // display name (e.g. 'Tropical Fish')
  description: string,
  layout: string[][],  // 2D grid of color strings and nulls
  bench: { color: string, ammo: number }[]
}
```

This file is imported by `src/data/levels.js` and appended to the level list.

### CLI Interface

```bash
# Generate all 50 levels
ANTHROPIC_API_KEY=sk-... node tools/generator/generate-pixel-art-levels.js

# Generate a single level
ANTHROPIC_API_KEY=sk-... node tools/generator/generate-pixel-art-levels.js --level 51

# Preview in terminal (ASCII grid)
ANTHROPIC_API_KEY=sk-... node tools/generator/generate-pixel-art-levels.js --level 51 --preview

# Custom output path
ANTHROPIC_API_KEY=sk-... node tools/generator/generate-pixel-art-levels.js --out src/data/pixel-art-levels.js
```

## Sprite Assets

10 new textures needed (2 colors × 5 per color):
- `cube-{color}-top` — cube face texture
- `pig-{color}` — front-facing pig
- `pig-{color}-back` — back-facing pig
- `pig-{color}-left` — left-facing pig
- `pig-{color}-right` — right-facing pig

Colors: orange, purple. Initial implementation generates these programmatically by tinting existing sprites (BootScene already has createPigTexture and createTopCubeTexture helpers). Art can be refined later with custom sprites.

## Out of Scope

- Replacing existing levels 1–50
- New milestone/reward system
- Mobile UI changes for larger grids (existing responsive scaling handles up to 12×12)
- Custom pig animations per color
- Sound effects for new colors
